- Define CalculatorService interface with:

- Implement a struct satisfying the interface.
- Use switch-case initially.
- Support:
  - ADD
  - SUBTRACT
  - MULTIPLY
  - DIVIDE

- Create calculator-back/internal/service/calculator_service_test.go.
- Use Testify.
- Test:
  - Add
  - Subtract
  - Multiply
  - Divide
  - Division by zero
  - Invalid operation
- Keep logic isolated from HTTP.
- Do not use frameworks in the service layer.

3. Step 3: Refactor Service to Strategy Pattern

Goal:

- Remove the switch-case from the calculator service.
- Use a map of operations to strategy functions.
- Follow the Open/Closed Principle.

Requirements:

- Replace switch-case with:
  map[domain.Operation]func(request Request) (float64, error)
- Keep the CalculatorService interface unchanged.
- Move Request into calculator_service.go.
- Use Request to aggregate operation parameters.
- Preserve existing behavior.
- Do not break existing tests.
